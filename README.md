# Francesco Agnoletto Portfolio Dashboard

This repository contains a portfolio dashboard built with a zero-dependency frontend and a small AWS Lambda backend. The goal is to showcase a cost-effective architecture with minimal complexity, secure deployment, and automated metric generation.

## Project overview

- Frontend: TypeScript + Vite + Web Components.
- Backend: two AWS Lambda functions that generate CloudFront metrics and AWS Cost Explorer data.
- Deployment: GitHub Actions deploys the frontend to Amazon S3 and CloudFront, and updates each Lambda independently.
- AWS access: OIDC-based GitHub Actions role assumption, with no long-lived API keys stored in the repo.

## Architecture

1. Code is pushed to GitHub.
2. `frontend-deploy.yml` runs on frontend-related changes and builds the browser app.
3. The frontend bundle is uploaded to a private S3 bucket.
4. CloudFront is invalidated for the site entry points.
5. Two server workflows build and deploy Lambda artifacts separately:
   - `server-deploy-cloudwatch-lambda.yml` for the CloudWatch metrics lambda.
   - `server-deploy-cost-explorer-lambda.yml` for the Cost Explorer lambda.
6. Each Lambda writes its output JSON to S3, where the site loads it at runtime.

## Frontend

The frontend is a lightweight dashboard rendered using custom elements and no runtime UI framework.

- Entry point: `src/main.ts`
- Web components: `src/web-components/`
- Styles: `src/styles/`
- Client state & data loading: `src/store/`
- Data displayed: deploy stats, CloudFront metrics, and cost metrics.

The site uses Vite for development and build tooling.

## Backend

The `server/` package contains the Lambda code.

- `server/src/cloudwatch-metrics-lambda.ts`
  - Queries CloudFront metrics from CloudWatch.
  - Calculates aggregate values for requests, bandwidth, cache hit rate, error rate, and origin latency.
  - Uploads `data/cloudfront-metrics.json` to S3.

- `server/src/cost-explorer-metrics-lambda.ts`
  - Queries AWS Cost Explorer for the current month.
  - Produces a service-level cost breakdown.
  - Uploads `data/cost-metrics.json` to S3.

Both lambdas use the AWS SDK v3 and write JSON output to the configured S3 bucket.

## Local development

At the repository root:

```bash
npm install
npm run dev
```

Build and preview locally:

```bash
npm run build
npm run preview
```

Run TypeScript checks:

```bash
npm run typecheck
```

## Server package

The server package is versioned separately and contains its own `package.json`.

Build the Lambda bundles:

```bash
cd server
npm install
npm run build-cloudwatch
npm run build-cost-explorer
```

Each script bundles its handler with `esbuild` targeting Node 24.

## Deployment workflows

The repository includes three GitHub Actions workflows:

- `.github/workflows/frontend-deploy.yml`
  - Triggers on changes to `src/**`, `public/**`, `index.html`, `error.html`, root `package.json`, `vite.config.ts`, or `tsconfig.json`.
  - Runs `npm ci`, type checks, builds the frontend, and syncs `dist/` to S3.
  - Invalidates CloudFront for `/`, `/index.html`, and `/data/deploy-stats.json`.

- `.github/workflows/server-deploy-cloudwatch-lambda.yml`
  - Triggers on changes to `server/src/cloudwatch-metrics-lambda.ts`, `server/src/utils/*`, or `server` package metadata.
  - Builds the CloudWatch lambda and deploys it to the configured Lambda function.

- `.github/workflows/server-deploy-cost-explorer-lambda.yml`
  - Triggers on changes to `server/src/cost-explorer-metrics-lambda.ts`, `server/src/utils/*`, or `server` package metadata.
  - Builds the Cost Explorer lambda and deploys it to the configured Lambda function.

## Output data files

The deployed site consumes JSON data generated by the backend lambdas:

- `data/cloudfront-metrics.json`
- `data/cost-metrics.json`
- `data/deploy-stats.json`

These files are uploaded to S3 and consumed by the frontend dashboard.

## Notes

- The frontend is intentionally minimal and framework-free.
- AWS credentials are provided via GitHub Actions OIDC and role assumption.
- The site is designed for a secure, low-complexity portfolio deployment with periodic server-side metric updates.
